实验器材:
	GD32F427ZGT6核心板 V1.1
	
实验目的:
	学习GD32的IAP功能,掌握IAP的使用.
	
硬件资源:
	1,LED0(连接在PF9)
	2,串口1(波特率:460800,PA9/PA10连接在板载USB转串口芯片CH340上面)
    3,4.3/5.0/7.0寸WKS显示模块(通过核心板的两个LCD接口驱动)
	4,按键KEY0(PE4)/WK_UP(PA0)
	
实验现象:
	本实验开机的时候先显示提示信息，然后等待串口输入接收APP程序（无校验，一次性接收），在串口接收
	到APP程序之后，即可执行IAP。如果是SRAM APP，通过按下KEY0即可执行这个收到的SRAM APP程序。如果
	是FLASH APP，则需要先按下KEY_UP按键，将串口接收到的APP程序存放到GD32的FLASH，之后再按KEY0即
	可以执行这个FLASH APP程序。

注意事项: 
	1,根据所使用的LCD接口，在本例程的if_port.h文件里选择相应的驱动方式(FSMC/4线SPI)。
	2,本实验下载成功后,需用串口调试助手发送SRAM APP/FLASH APP等APP代码(.bin文件)验证IAP功能.
	3,SRAM APP代码的起始地址必须是：0X20001000，FLASH APP代码的起始地址必须是：0X08008000。
	4,在创建APP代码的时候，切记要设置中断向量偏移量！！
